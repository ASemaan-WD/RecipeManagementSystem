# UI Component Skill

> This skill defines the canonical conventions for shadcn/ui primitive components. These components live in `src/components/ui/` and are typically generated by the shadcn CLI, but may be manually edited or extended. This skill documents the patterns to follow when modifying or creating UI primitives.

---

## When This Applies

- **Modifying** an existing shadcn/ui component (adding variants, fixing styles)
- **Creating** a new UI primitive not available from shadcn (rare)
- **Understanding** the conventions when consuming these components elsewhere

Most UI components are added via the CLI:

```bash
npx shadcn@latest add button
npx shadcn@latest add card
```

Manual creation should be rare — prefer the CLI and then modify if needed.

---

## File Naming & Location

- **Naming:** `kebab-case.tsx` (e.g., `button.tsx`, `card.tsx`, `alert-dialog.tsx`, `dropdown-menu.tsx`)
- **Location:** `src/components/ui/`
- **One component family per file** — a file may export multiple related sub-components (compound pattern).

---

## Import Ordering (Mandatory)

```typescript
// 1. React
import * as React from 'react';

// 2. Third-party libraries (CVA, Radix, etc.)
import { cva, type VariantProps } from 'class-variance-authority';
import { Slot } from 'radix-ui';

// 3. Internal utilities
import { cn } from '@/lib/utils';
```

## Component Patterns

### Simple Wrapper Component

The simplest pattern — wraps an HTML element with default styling:

```typescript
import * as React from "react"

import { cn } from "@/lib/utils"

function Skeleton({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="skeleton"
      className={cn("bg-accent animate-pulse rounded-md", className)}
      {...props}
    />
  )
}

export { Skeleton }
```

### Variant Component (CVA)

For components with multiple visual variants:

```typescript
import * as React from "react"
import { cva, type VariantProps } from "class-variance-authority"
import { Slot } from "radix-ui"

import { cn } from "@/lib/utils"

const buttonVariants = cva(
  "inline-flex items-center justify-center gap-2 whitespace-nowrap rounded-md text-sm font-medium transition-all disabled:pointer-events-none disabled:opacity-50",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        destructive: "bg-destructive text-white hover:bg-destructive/90",
        outline: "border bg-background shadow-xs hover:bg-accent hover:text-accent-foreground",
        secondary: "bg-secondary text-secondary-foreground hover:bg-secondary/80",
        ghost: "hover:bg-accent hover:text-accent-foreground",
        link: "text-primary underline-offset-4 hover:underline",
      },
      size: {
        default: "h-9 px-4 py-2",
        sm: "h-8 rounded-md px-3",
        lg: "h-10 rounded-md px-6",
        icon: "size-9",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
)

function Button({
  className,
  variant = "default",
  size = "default",
  asChild = false,
  ...props
}: React.ComponentProps<"button"> &
  VariantProps<typeof buttonVariants> & {
    asChild?: boolean
  }) {
  const Comp = asChild ? Slot.Root : "button"

  return (
    <Comp
      data-slot="button"
      data-variant={variant}
      data-size={size}
      className={cn(buttonVariants({ variant, size, className }))}
      {...props}
    />
  )
}

export { Button, buttonVariants }
```

### Compound Component Pattern

For components composed of multiple sub-components:

```typescript
import * as React from "react"

import { cn } from "@/lib/utils"

function Card({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card"
      className={cn(
        "bg-card text-card-foreground flex flex-col gap-6 rounded-xl border py-6 shadow-sm",
        className
      )}
      {...props}
    />
  )
}

function CardHeader({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card-header"
      className={cn("grid auto-rows-min gap-2 px-6", className)}
      {...props}
    />
  )
}

function CardTitle({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card-title"
      className={cn("leading-none font-semibold", className)}
      {...props}
    />
  )
}

function CardContent({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card-content"
      className={cn("px-6", className)}
      {...props}
    />
  )
}

function CardFooter({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      data-slot="card-footer"
      className={cn("flex items-center px-6", className)}
      {...props}
    />
  )
}

export { Card, CardHeader, CardTitle, CardContent, CardFooter }
```

## Pattern Details

### Props Typing

- Always use `React.ComponentProps<"element">` — NOT custom interfaces.
- Spread `...props` onto the rendered element for full HTML attribute passthrough.
- Always accept `className` and merge with `cn()`.

```typescript
function Input({ className, ...props }: React.ComponentProps<"input">) {
  return (
    <input
      data-slot="input"
      className={cn("base-classes", className)}
      {...props}
    />
  )
}
```

### `data-slot` Attribute

Every UI component renders a `data-slot` attribute for styling hooks:

```typescript
<div data-slot="card" ...>
<button data-slot="button" ...>
<input data-slot="input" ...>
```

- Value is the component name in `kebab-case`.
- Enables parent components to style children via `has-data-[slot=card-action]` selectors.

### `data-variant` and `data-size` Attributes

Components with variants also render their current variant as data attributes:

```typescript
<button data-slot="button" data-variant={variant} data-size={size} ...>
```

### `asChild` Pattern (Radix Slot)

Components that support rendering as a different element use the `asChild` prop:

```typescript
import { Slot } from "radix-ui"

function Button({ asChild = false, ...props }) {
  const Comp = asChild ? Slot.Root : "button"
  return <Comp data-slot="button" {...props} />
}

// Usage: render Button styling on a Link
<Button asChild>
  <Link href="/path">Click me</Link>
</Button>
```

### Variant Definition (CVA)

- Define variants as a `const` above the component.
- Export the variants object alongside the component (e.g., `buttonVariants`).
- Use `defaultVariants` to set sensible defaults.

### `cn()` Usage

Always use `cn()` from `@/lib/utils` to merge classes:

```typescript
className={cn("base-classes", className)}
className={cn(buttonVariants({ variant, size, className }))}
```

### Exports

- **Named exports only** — group in a single `export { }` statement at the bottom.
- Export the component(s) and optionally the variants object.
- **No default exports**.

```typescript
export { Button, buttonVariants };
export { Card, CardHeader, CardTitle, CardContent, CardFooter };
export { Skeleton };
```

### No `'use client'`

Most UI primitives are server-renderable. Only add `'use client'` if the component uses:

- React hooks (dropdown menus, dialogs, etc. that use Radix state)
- Event handlers

Simple wrapper components (Card, Skeleton, Badge) do NOT need `'use client'`.

## Anti-Patterns

- **Never** use custom interfaces for props — use `React.ComponentProps<"element">`.
- **Never** skip `data-slot` — it's used for parent-child styling hooks.
- **Never** skip `cn()` for className merging — consumer overrides depend on it.
- **Never** use `export default` — always named exports.
- **Never** add business logic to UI primitives — they are pure presentational.
- **Never** import domain-specific types — UI components are generic.
- **Never** hardcode colors — use Tailwind CSS theme variables (`bg-primary`, `text-muted-foreground`).
- **Never** modify the core pattern of CLI-generated components unless adding new variants — updates via CLI may overwrite changes.

---

## Checklist

Before committing a UI component change:

- [ ] File is `kebab-case.tsx` in `src/components/ui/`
- [ ] Props use `React.ComponentProps<"element">`
- [ ] `className` is accepted and merged with `cn()`
- [ ] `data-slot` is set on the root element
- [ ] CVA components export the variants object
- [ ] Compound components export all sub-components
- [ ] Named exports at the bottom of the file
- [ ] No business logic or domain-specific imports
- [ ] Colors use Tailwind theme variables (not hardcoded hex/rgb)
- [ ] `'use client'` only if truly needed (hooks or Radix state)
